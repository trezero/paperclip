# Marketing Hub — Design Specification

**Date:** 2026-03-29
**Status:** Draft
**Approach:** Hybrid (Plugin UI + Minimal Host Bridge Extensions)

## Overview

A unified marketing workspace inside Paperclip, built as a BrandAmbassador plugin page with three targeted host bridge extensions. The hub consolidates content pipeline management, tool execution, trend discovery, and engagement monitoring into a single cohesive interface where humans and AI agents collaborate as equals.

## Architecture

### Approach: Hybrid (Plugin UI + Host Bridge Extensions)

The hub UI lives entirely in the BrandAmbassador plugin bundle, rendered via the existing `page` slot type. The Paperclip host receives three small, generic bridge additions that unlock interactive tool execution for any plugin.

**Why this approach:**
- Self-contained: install the plugin, get the hub; uninstall, it's gone
- Minimal host changes — three well-scoped bridge additions that benefit all future plugins
- No speculative framework-building; each host change solves a concrete need
- Plugin bridge already supports data fetching, actions, and real-time streaming

## Part 1: Paperclip Host Bridge Extensions

Three additions to the plugin bridge in the Paperclip host. All are generic and usable by any plugin.

### 1.1 `executeTool` Bridge Method

Lets a plugin's UI invoke any of that plugin's own registered tools (scoped to calling plugin for security).

**Plugin UI usage:**
```ts
usePluginAction("execute-tool", {
  toolName: "generate-caption",
  parameters: { topic: "ETH staking", platform: "twitter" },
  runContext: { companyId, agentId? }
})
```

**Host implementation:** New action handler in `server/src/routes/plugins.ts` that:
- Receives tool name and parameters from the bridge
- Namespaces the tool name with the calling plugin's key (`generate-caption` → `brandambassador.marketing:generate-caption`)
- Delegates to existing `toolDispatcher.executeTool()` — same path agents use
- Returns the `ToolResult` to the plugin UI

This is a thin adapter — no new execution logic.

### 1.2 `navigateToEntity` Bridge Hook

Lets plugin UI navigate to Paperclip entities without knowing host routes.

**Plugin UI usage:**
```ts
hostContext.navigateToEntity({
  type: "issue", id: "MF-147", companyId
})
```

**Host implementation:** New function added to the `useHostContext()` return value in `ui/src/plugins/bridge.ts`. The host bridge translates entity type + ID into the correct host route and calls `navigate()` from the host router. Enables bidirectional linking between Kanban cards and Paperclip issues.

### 1.3 `getToolSchemas` Bridge Data Method

Lets plugin UI fetch its own tools' `parametersSchema` definitions for auto-form generation.

**Plugin UI usage:**
```ts
usePluginData("tool-schemas")
```

**Host implementation:** New data handler that returns the manifest's `tools[]` array with schemas for the calling plugin. Plugin UI uses these to build input forms dynamically.

## Part 2: Marketing Hub UI (BrandAmbassador Plugin)

### 2.1 Layout Structure

Three-zone layout: left sidebar, main canvas, contextual right panel.

**Plugin registration:** One `page` slot at route `/:companyPrefix/marketing`. The hub manages its own internal navigation via React state (no host routing needed).

**Components:**
- `MarketingHub` — root layout with sidebar + canvas + right panel
- `Sidebar` — navigation sections + persistent brand context panel
- `CanvasRouter` — swaps main canvas content based on active section
- `DetailPanel` — contextual right panel, slides open/closed

**Sidebar sections:**
- Pipeline (primary)
- Create
- Discover
- Monitor
- Brand context footer (always visible)

### 2.2 Pipeline View (Primary Workspace)

Kanban board with four columns: **Draft → Review → Approved → Published**.

**Content card data model:**
- `id` — unique identifier (UUID, generated by plugin)
- `topic` — content subject
- `platform` — target platform (twitter, telegram, reddit, instagram, tiktok, youtube)
- `caption` — generated or manual caption text
- `mediaRef` — filename of generated image/video (if any)
- `mediaType` — "image" or "video"
- `moderationScore` — 0–100 safety score
- `status` — draft | review | approved | published
- `source` — "human" | "agent" (with agent ID if agent-created)
- `linkedIssueId` — Paperclip issue ID (created when card enters Review)
- `createdAt`, `updatedAt` — timestamps

**Status transitions (v1 — button-based, not drag-and-drop):**
- Draft → Review: creates a Paperclip issue via existing `queue-post` action, links bidirectionally
- Review → Approved: inline Approve button in detail panel, or Paperclip issue resolution triggers `queue-process` job
- Review → Draft: Reject button sends back for revision
- Approved → Published: manual or agent-driven publish step

**Card detail (right panel):** Opens when a card is clicked. Shows:
- Media preview (image/video rendered inline)
- Caption text
- Moderation score and brand-voice compliance
- Link to Paperclip issue (via `navigateToEntity` bridge hook)
- Action buttons: Approve, Reject, Edit, Regenerate

**Header actions:**
- "+ New Post" — jumps to Create workshop with blank slate
- "Auto-Generate" — triggers `generate-post` tool via `executeTool` bridge with brand settings pre-filled

### 2.3 Create View (Workshop)

Chained tool execution workspace for manually building content.

**Layout:**
- **Left column** — Tool palette: all 7 tools as compact cards. Click to activate.
- **Center** — Active tool workspace: parameter form (auto-generated from `parametersSchema` via `getToolSchemas`) + "Run" button. Results render inline below form after execution.
- **Right column** — Chain context: accumulated results from previous tool runs in this session (caption, media, moderation score, knowledge context used). Builds up the pieces of a post.

**Chaining flow (optional, not mandatory):**
1. `check-trends` → results appear → user clicks a trend
2. Trend topic auto-fills `generate-caption` form → run → caption in chain context
3. Caption auto-fills `generate-media` prompt → run → live progress via `usePluginStream` (`generation-progress` channel) → image preview in chain context
4. Caption auto-fills `moderate-content` → run → moderation result in chain context
5. Chain context has topic + caption + media + moderation → "Create Post" button calls a new `create-pipeline-card` plugin action handler that assembles the chain context into a content card and writes it to plugin state with status `draft`

Each tool also works standalone — chaining is optional context feeding.

**Media generation UX:**
- Progress indicator during `generate-media` / `generate-post` execution
- Plugin subscribes to `generation-progress` stream channel via `usePluginStream`
- Progress events update a bar (queued → generating → complete)
- On completion, image/video renders inline as preview

**Contextual brand overrides:**
- `generate-caption` form shows current tone as pre-filled dropdown, user can override per-run
- `generate-media` shows platform-specific dimensions pre-selected from brand settings, overridable
- Overrides are per-execution only — saved brand settings unchanged

### 2.4 Discover View

Trend list from `check-trends` tool results (stored in plugin state `trends:active`).

**Data source:** `usePluginData("trends")`

**UI:**
- Filter bar: platform, minimum score, keyword search
- Trend cards showing: title, platform, score badge, sentiment, growth indicator
- **"Create from this"** button → jumps to Create workshop with topic pre-filled
- **"Dismiss"** button → calls existing `dismiss-trend` action

### 2.5 Monitor View

Published post engagement overview from `engagement-check` job results.

**Data source:** `usePluginData("engagement")`

**UI:**
- Cards showing published posts with engagement metrics (likes, RTs, comments, upvotes)
- Sorted by engagement — highest-performing content at top
- **"Create follow-up"** button → jumps to Create workshop with original topic as context
- **"View post"** → opens Kanban card detail for that published post

### 2.6 Brand Context (Contextual Inline)

No dedicated settings page. Brand configuration appears where needed.

**Sidebar footer (always visible):**
- Current brand name, tone, target audience, active platforms
- "Edit" link → inline accordion expand with form fields
- Saves via existing `save-brand-settings` action handler
- When settings change, tool forms referencing brand context update automatically

**Brand Knowledge subsection (in sidebar footer):**
- Document count and "Upload Document" button (calls existing `upload-document` action)
- No search box, no document browser — the knowledge base is backend infrastructure
- When tools retrieve knowledge context, it shows as "Context used" in the Create workshop's chain context panel

## Part 3: Data Flow & Integration

### Content Lifecycle

```
Trend discovered (background job)
  → User clicks "Create from this" OR Agent auto-generates
    → Tools execute in chain (caption → media → moderation)
      → Content card created in plugin state (status: draft)
        → User moves to Review via detail panel
          → Plugin creates Paperclip issue (bidirectional link)
            → Issue resolved in Paperclip → queue-process job advances card
              → Card moves to Approved → Published
```

### State Ownership

| Data | Storage | Access Method |
|------|---------|---------------|
| Content cards | Plugin state (per-company) | `usePluginData("pipeline")` |
| Trends | Plugin state (`trends:active`) | `usePluginData("trends")` |
| Engagement metrics | Plugin state (`engagement:conversations`) | `usePluginData("engagement")` |
| Brand settings | Plugin state (`brand:settings`) | `usePluginData("brand-settings")` |
| Approval status | Paperclip issues | Bridge `navigateToEntity` for linking |
| Media files | Disk (ComfyUI output) | Referenced by filename in content cards |
| Knowledge base docs | Archon RAG store | Backend-only, tools search automatically |

### Real-Time Streams

| Channel | Purpose | Consumer |
|---------|---------|----------|
| `generation-progress` | ComfyUI generation progress | Create workshop progress bar |
| `comfyui-status` | GPU health | Small status indicator in Create view |
| `trends-updated` | New trends discovered | Discover view refresh |
| `content-published` | Post published | Pipeline Kanban card movement |

All consumed via `usePluginStream` — no host changes needed.

### Agent ↔ Human Visibility

- Content cards tagged with `source: "agent"` or `source: "human"`
- Kanban board shows both with distinct visual indicators
- Agent activity appears in Paperclip activity feed via existing `activity` API
- Both humans and agents create content through the same tool execution pipeline

## Part 4: Scope

### V1 Deliverables

1. Three Paperclip host bridge additions (`executeTool`, `navigateToEntity`, `getToolSchemas`)
2. Marketing Hub page registered as BrandAmbassador plugin page slot
3. Sidebar + Canvas + Detail Panel layout with 4 sections
4. Pipeline Kanban — 4 columns, card detail panel, button-based status transitions, Paperclip issue linking
5. Create Workshop — tool palette, auto-generated parameter forms, chained execution, live media preview
6. Discover View — trend list with filters and "Create from this" action
7. Monitor View — published post list sorted by engagement with "Create follow-up" action
8. Contextual brand settings in sidebar footer with inline editing
9. Agent/Human attribution on all content cards

### V1 Explicit Exclusions

- No drag-and-drop Kanban (button-based transitions only)
- No campaign grouping (all content in one pipeline)
- No scheduling (Approved → Published is manual/agent-driven)
- No actual platform publishing (cards marked "published" but not posted)
- No real trend sources (renders whatever `trend-scan` job provides)
- No real engagement data (renders whatever `engagement-check` job provides)
- No charts or analytics dashboards
- No Knowledge Base UI (backend-only, documents uploaded via brand settings)

## Part 5: V2 Roadmap

Built immediately after v1 validation. Items grouped by priority.

### High Priority (Immediately After V1)

**1. Drag-and-Drop Kanban**
- Proper drag between columns with visual drop targets
- Drag triggers the same status transition logic as button clicks
- Maintains Paperclip issue creation/linking on Review entry

**2. Campaign Management**
- Campaign entity: name, date range, description, target platforms
- Content cards belong to a campaign (or "uncategorized")
- Pipeline view filterable by campaign
- Campaign summary: card counts by status, overall progress

**3. Content Scheduling**
- Set publish date/time on approved content
- Calendar view alongside or replacing the Pipeline Kanban (toggle)
- `queue-process` job checks scheduled times before marking as published
- Visual indicators on cards: scheduled time, countdown to publish

### Medium Priority (As Platform Integrations Land)

**4. Platform Publishing**
- Wire `queue-process` to post to Twitter, Telegram, Reddit via platform APIs
- OAuth token management per platform (stored in plugin secrets)
- Platform-specific formatting (thread splitting for Twitter, markdown for Telegram)
- Publish confirmation with link to live post

**5. Real Trend Sources**
- Connect `trend-scan` to RSS feeds, social API endpoints
- Integration with Researcher agent (if available) for deeper analysis
- Trend scoring improvements: velocity, relevance decay, competitor mentions
- Configurable trend sources per brand

**6. Real Engagement Data**
- Wire `engagement-check` to platform APIs for actual metrics
- Pull likes, retweets, comments, mentions, impressions
- Attribution: link engagement back to specific published content cards
- Notification when engagement thresholds hit (viral detection)

### Polish & Power Features

**7. Analytics Dashboard**
- Charts: engagement over time, best-performing content types, optimal posting times
- Platform comparison: which platform drives most engagement
- Agent vs. human performance comparison
- Content type breakdown (image vs. text vs. video performance)

**8. Content Templates**
- Save successful post formats as reusable templates
- Template variables (topic, platform, tone) auto-filled on use
- Template library browsable in Create workshop
- Agent can use templates for consistency

**9. Bulk Operations**
- Multi-select cards in Kanban view
- Batch approve, batch reject, batch move
- Batch regenerate (re-run tools on multiple cards)

**10. A/B Testing**
- Generate multiple caption/media variants for same topic
- Track which variant performs better after publishing
- Variant comparison view with engagement metrics
- Feed winning patterns back into brand settings
